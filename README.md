# 領藥（調劑）日期查詢工具

由健保調劑申報檔 `DRUGT.xml` 產生一個**單一 HTML 離線查詢頁**：輸入身分證字號，即可列出每次領藥（調劑）日期、就醫日、科別、院所、以及藥品明細（用量／用法／途徑／日數／總量）。申報檔僅記錄到「日」，無時:分。

## 特性

- **不含個資**：產生的 HTML 不嵌入任何病患資料。開頁後由使用者自行「匯入查詢來源」選擇 `DRUGT.xml` 或 `DRUGT.zip`，於瀏覽器端即時解析（自動處理 Big5 編碼與 zip 解壓）。
- **領藥時間**：申報檔僅到「日」。另可「匯入領藥時間檔」選擇每日領藥紀錄檔（如 `S1150618.txt`，Big5 制表符分隔），補上每次領藥的精確**時:分**。可一次多選、跨日累積匯入，查詢時與申報檔自動合併並去重；日期未在申報檔者，會直接以每日明細呈現（標示「每日明細」）。慢箋提前領藥時，另標示「實際領藥」日期。
- **記住來源**：匯入後存於本機瀏覽器 IndexedDB，下次開頁自動載入；選新檔即覆蓋，另有「清除預設」「清除時間檔」。
- **藥名 / 院所對照**：建置時把公開參考資料（健保用藥代碼→中文藥名、院所代碼→名稱）嵌入頁面。

## 使用方式

1. 將參考檔放入本資料夾（皆未隨本專案上傳）：
   - `健保用藥_YYYY-MM 歷史資料.xlsx`（藥名對照，必要）
   - `健保特約醫療院所名冊*.csv`（院所對照，可選；下載：https://data.gov.tw/dataset/168341 ）
2. 執行：

   ```bash
   python build.py
   ```

   （Windows 可雙擊 `重建查詢頁.bat`）
3. 產生 `領藥查詢.html`，用瀏覽器打開，匯入 `DRUGT.xml`/`DRUGT.zip` 後即可查詢；如需精確領藥時間，再用「⏰ 匯入領藥時間檔」選擇每日的 `S*.txt`（可多選累積）。

### 每日領藥紀錄檔（S 檔）欄位

每日 `S*.txt` 為 Big5、制表符分隔，每列一項藥品；自由文字欄位內含換行，解析時以行首 `處方調劑` 為每筆起點。主要使用欄位（0 起算）：

| 欄 | 意義 | 欄 | 意義 |
|----|------|----|------|
| 2  | 處方箋流水號（同箋聚合） | 35 | 藥品代碼 |
| 3  | 處方醫療院所代碼（對 d21） | 36/81 | 英文／中文藥名 |
| 4  | 調劑（申報）日期 | 65 | 處方院所名稱 |
| 5  | 就醫日期 | 69 | 科別名稱 |
| 6  | 身分證號（對 d3） | 92/93/94 | 處方建立／調劑／領藥時間戳（民國 YYYMMDDHHMMSS） |
| 7  | 姓名 | 59 | 出生（民國） |

領藥時:分取欄 92/93/94 中最晚的時間戳（最後一關＝實際交付）。

## 隱私

- `DRUGT.xml` / `DRUGT.zip` 為個人健康資料，**請勿上傳或外流**，已列入 `.gitignore`。
- 參考用 Excel／CSV 與產生的 HTML 亦不納入版控。
- 申報檔解析全程在本機瀏覽器內完成，不會上傳至任何伺服器。
